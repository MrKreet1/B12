# Большой итоговый отчет по расчетам кластера B12

## Название работы

Многостадийный поиск низкоэнергетических структур нейтрального кластера B12 методом многостартового скрининга `GFN2-xTB` с последующим уточнением на уровне `r2SCAN-3c` и проверкой колебательной устойчивости расчетом `NumFreq` в ORCA 6.1.

## Аннотация

В работе выполнен расчетный поиск устойчивых структур нейтрального кластера B12. Для уменьшения зависимости результата от одной стартовой геометрии использован многостартовый протокол: сначала сгенерирован набор регулярных и случайных начальных структур, затем для каждой из них проведена предварительная оптимизация методом `GFN2-xTB` для двух мультиплетностей, после чего результаты были отфильтрованы по признакам нормального завершения, реальной сходимости оптимизации и структурной уникальности. Отобранные низкоэнергетические кандидаты были переоптимизированы методом `r2SCAN-3c`, а затем проверены расчетом `NumFreq`.

По результатам расчетов установлено, что наименьшую энергию имеет синглетная структура `double_ring_6_6_d1.90_m1_r2scan` с энергией `-297.805239027148 Eh`. Вторая устойчивая структура `ring12_d1.70_m1_r2scan` также соответствует истинному минимуму, но расположена выше на `632.10 кДж/моль`. Триплетная структура `zigzag_chain12_d1.50_m3_r2scan` не проходит частотную проверку, поскольку имеет `6` мнимых частот, из которых `5` лежат ниже порога `-20.0 см^-1`, а минимальная частота составляет `-2022.99 см^-1`. В пределах выполненного расчетного протокола устойчивыми минимумами являются только две синглетные структуры, а двухкольцевой мотив `double_ring_6_6` является наиболее выгодным.

## 1. Цель и задачи расчета

Целью работы было определить наиболее устойчивую структуру B12 в исследованном наборе стартовых геометрий и мультиплетностей, а также отделить истинные минимумы поверхности потенциальной энергии от неустойчивых стационарных точек.

Для достижения этой цели были решены следующие задачи:

1. сгенерирован набор регулярных и случайных стартовых геометрий B12;
2. выполнен предварительный массовый скрининг методом `GFN2-xTB`;
3. проведена фильтрация по завершению расчета, сходимости оптимизации и структурной уникальности;
4. выполнено DFT-уточнение лучших кандидатов методом `r2SCAN-3c`;
5. выполнен частотный анализ `NumFreq` для проверки устойчивости найденных структур;
6. составлен финальный ранжированный список структур и выделены формы без мнимых частот.

## 3. Вычислительный протокол

### 3.1. Генерация стартовых геометрий

Стартовые структуры создавались скриптом `pipeline.py make-starts`. Использовались три регулярных мотива:

- `ring12`
- `double_ring_6_6`
- `zigzag_chain12`

Кроме регулярных мотивов, для каждого стартового расстояния были сгенерированы случайные компактные структуры с контролем минимального межатомного расстояния. В итоге было получено:

- `12` регулярных стартов: `3` мотива `x 4` расстояния;
- `40` случайных стартов: `10` случайных структур `x 4` расстояния;
- всего `52` стартовые геометрии.

### 3.2. Предварительный xTB-скрининг

Для каждой из `52` стартовых геометрий были подготовлены входные файлы вида:

`! GFN2-xTB TightSCF TightOpt`

Поскольку расчеты проводились для двух мультиплетностей (`1` и `3`), суммарное число xTB-задач составило:

`52 x 2 = 104`.

На этом этапе использовались `1` процесс и `500 MB` памяти на задачу, что позволяло выполнять широкий скрининг без существенной вычислительной нагрузки.

### 3.3. Критерии отбора xTB-результатов

После завершения xTB-этапа скрипт `pipeline.py rank-xtb` извлекал:

- финальную энергию;
- финальные координаты;
- признак нормального завершения расчета;
- признак сходимости геометрической оптимизации.

Для дальнейшего анализа принимались только те результаты, у которых одновременно выполнялись условия:

1. расчет завершился нормально;
2. энергия была успешно извлечена;
3. геометрическая оптимизация была сошедшейся;
4. финальные координаты присутствовали и могли быть записаны в итоговый файл.

### 3.4. Удаление структурных дублей

Дубликаты распознавались по fingerprint-представлению, построенному на основе отсортированных попарных межъядерных расстояний. Если RMS-отклонение между двумя такими fingerprint-векторами было меньше `0.05 A`, структуры считались эквивалентными, и в итоговый набор сохранялся только более низкоэнергетический представитель.

Этот шаг особенно важен для многостартовых расчетов, поскольку одна и та же геометрическая форма может воспроизводиться из разных стартовых расстояний и из разных исходных конфигураций.

### 3.5. DFT-уточнение

Для низкоэнергетических уникальных кандидатов автоматически формировались входные файлы:

`! r2SCAN-3c TightSCF TightOpt`

DFT-этап выполнялся с параметрами `nprocs = 8` и `MaxCore = 2000 MB`. Формально настройка позволяла отправить на DFT до `10` лучших xTB-структур, однако фактически после фильтрации и удаления дублей пригодными оказались только `3` уникальных кандидата.

### 3.6. Частотная проверка

Для финальных DFT-структур были сформированы входные файлы:

`! r2SCAN-3c TightSCF NumFreq`

Из выходных файлов `NumFreq` извлекались:

- финальная энергия;
- число распознанных частот;
- минимальная частота;
- число отрицательных частот по строгому критерию `n_imag_strict`;
- число значимых отрицательных частот `n_imag_significant`, то есть частот ниже `-20.0 см^-1`.

Использовались следующие итоговые статусы:

| Статус | Интерпретация |
| --- | --- |
| `OK_NO_IMAG` | мнимых частот нет |
| `REJECT_IMAG` | присутствуют значимые мнимые частоты |

В данном проекте реализовались только эти два статуса.

### 3.7. Формула расчета относительных энергий

Для сравнения структур использовалась формула:

`dE_i = (E_i - E_min) * 2625.49962`,

где `E_i` и `E_min` выражены в Hartree, а коэффициент `2625.49962` переводит разность энергий в `кДж/моль`.

## 4. Сводная статистика расчетного конвейера

В `results/xtb_ranked_all.csv` присутствуют две технические записи `test_xtb_fix` и `test_xtb_fix2`, поэтому статистика основного xTB-прогона ниже рассчитывается по исходным `104` рабочим задачам.

| Этап | Число объектов | Комментарий |
| --- | ---: | --- |
| Сгенерированные стартовые геометрии | 52 | `12` регулярных + `40` случайных |
| Запланированные xTB-расчеты | 104 | `52` геометрии `x 2` мультиплетности |
| xTB-расчеты с нормальным завершением | 101 | `97.12%` от основного набора |

Эта статистика показывает, что многостартовый поиск действительно необходим: из `104` xTB-задач только `16` привели к сошедшимся минимумам, а после удаления дублей и энергетического отбора к DFT-этапу перешли только `3` структурных кандидата.

Таким образом, к DFT-этапу были переданы три структурно различных кандидата: двухкольцевой синглет, кольцевой синглет и зигзагообразный триплет. Триплетные варианты `double_ring_6_6` и `ring12` не вошли в итоговый список, поскольку соответствовали тем же геометрическим семействам, но были выше по энергии, чем синглетные представители.

## 7. Анализ полученных структур

### 7.1. Глобальный минимум: `double_ring_6_6_d1.90_m1_r2scan`

Структура `double_ring_6_6_d1.90_m1_r2scan` одновременно удовлетворяет трем ключевым условиям:

1. имеет наименьшую xTB-энергию среди уникальных кандидатов;
2. сохраняет первое место после DFT-уточнения;
3. не имеет мнимых частот по результатам `NumFreq`.

Следовательно, в пределах выполненного многостартового поиска и использованного уровня теории именно эта структура должна рассматриваться как наиболее вероятный глобальный минимум для исследованного набора B12.

### 7.2. Локальный минимум: `ring12_d1.70_m1_r2scan`

Структура `ring12_d1.70_m1_r2scan` также проходит частотную проверку и потому соответствует истинному локальному минимуму. Однако ее энергия выше энергии лидера на `632.10 кДж/моль`, что делает ее существенно менее выгодной по сравнению с `double_ring_6_6`.

Этот результат важен методически: кольцевой мотив не является расчетным артефактом и действительно существует как стационарное устойчивое решение, но по энергетике заметно проигрывает двухкольцевой форме.

### 7.3. Неустойчивая структура: `zigzag_chain12_d1.50_m3_r2scan`

Структура `zigzag_chain12_d1.50_m3_r2scan` после DFT-оптимизации не выдерживает частотную проверку:

- `n_imag_strict = 6`;
- `n_imag_significant = 5`;
- минимальная частота `-2022.99 см^-1`.

Такой набор мнимых частот нельзя интерпретировать как слабый численный шум. Это признак выраженной колебательной неустойчивости, то есть структура не соответствует истинному минимуму на поверхности потенциальной энергии. Поэтому она исключается из финального набора устойчивых форм.

## 9. Общий вывод по расчетам

Выполненный многостартовый расчетный протокол показал, что одиночного интуитивного старта для B12 недостаточно: из `104` xTB-задач лишь `16` дали сошедшиеся структуры, а после удаления геометрических дублей и финальной частотной проверки устойчивыми оказались только `2` формы. Это подтверждает, что задача поиска минимумов для B12 чувствительна к начальному приближению и требует каскадной схемы отбора.

На всем пути расчета - от xTB до DFT и `NumFreq` - наиболее выгодной остается структура `double_ring_6_6_d1.90_m1_r2scan`. Кольцевая структура `ring12_d1.70_m1_r2scan` подтверждается как локальный минимум, но располагается значительно выше по энергии. Триплетная зигзагообразная форма оказывается неустойчивой и должна быть исключена из числа финальных кандидатов.

Итоговый вывод состоит в том, что в пределах использованного набора стартов, двух рассмотренных мультиплетностей и уровня теории `r2SCAN-3c` наиболее стабильной структурой кластера B12 является двухкольцевой синглетный мотив `double_ring_6_6`.
